01

نظرة عامة على المشروع

مشروع Gas Delivery Platform هو منصة متكاملة لإدارة وتوصيل أسطوانات الغاز في سلطنة عُمان. يتكون المشروع من أربعة مكونات رئيسية تعمل معاً لتوفير تجربة شاملة لجميع الأطراف.

🖥️
لوحة تحكم
Next.js Admin Dashboard

لوحة تحكم ثنائية اللغة (عربي/إنجليزي) لإدارة الطلبات والسائقين والمناطق والمنتجات والإعدادات

⚙️
خادم API
Express + PostgreSQL

واجهة برمجية RESTful مع تحديثات لحظية عبر Socket.IO وقاعدة بيانات PostgreSQL

📱
تطبيق العميل
Flutter Customer App

تطبيق موبايل للعملاء لطلب الغاز وتتبع التوصيل مباشرة على الخريطة

🚗
تطبيق السائق
Flutter Driver App

تطبيق للسائقين لإدارة الطلبات والتوصيل والأرباح مع خرائط تفاعلية

نقاط تحتاج تحسين

!
عدم توحيد إصدار Riverpod

تطبيق العميل يستخدم flutter_riverpod ^3.0.3 بينما تطبيق السائق يستخدم ^2.6.1. يجب توحيد الإصدار لتقليل التعقيد.

!
غياب الاختبارات

لا توجد اختبارات للخادم (Express/Node.js). اختبارات Flutter مقتصرة على الحد الأدنى. يُوصى بإضافة Jest/Vitest للـ API واختبارات widget/integration لـ Flutter.

!
تعارض منفذ الخادم

.env.example يحدد PORT=3000 لكن server.js يستخدم 5000 كقيمة افتراضية. يجب توحيد القيمة.

!
غياب Rate Limiting

لا توجد حماية ضد هجمات إغراق الطلبات على API. يُوصى بإضافة express-rate-limit.

!
CORS مفتوح بالكامل

إعدادات CORS تسمح بأي مصدر (origin: true). يجب تقييد المصادر المسموحة في الإنتاج.

!
غياب تشفير كلمات المرور

لا توجد مكتبة bcrypt في التبعيات رغم وجود حقل password_hash. يجب التحقق من آلية التشفير.
